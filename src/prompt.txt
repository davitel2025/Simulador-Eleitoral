## CONTEXTO

Você vai trabalhar no projeto **Simulador Eleitoral** hospedado em `https://github.com/davitel2025/Simulador-Eleitoral`. É uma aplicação React + TypeScript + Vite + Tailwind CSS que simula eleições presidenciais brasileiras com mapa interativo, edição de votos por estado/município e geração de imagens de resultados.

### Estrutura de Arquivos Relevante

```
src/
├── App.tsx                                    # Componente raiz, gerencia fluxo de telas
├── types.ts                                   # Tipos: Candidate, PoliticalScenario, ElectionRound, etc.
├── data/
│   ├── scenarios.ts                           # Cenários históricos (2018, 2022) e projeções 2026
│   └── states.ts                              # Dados dos estados (UF, nome, eleitores, etc.)
├── data_cenarios/
│   ├── primeiro_turno_2026.json               # Dados JSON do 1º turno 2026
│   └── segundo_turno_2026.json                # Dados JSON do 2º turno 2026
├── lib/
│   ├── constants.ts                           # DEFAULT_COLORS, UF_NAME_MAP, etc.
│   ├── utils.ts                               # readFileAsBase64, normalizeVotesForCandidates, etc.
│   ├── color.ts                               # Utilidades de cor
│   └── geo.ts                                 # Projeções D3-geo
├── components/
│   ├── screens/
│   │   ├── InitialScreen.tsx                  # Tela inicial (seleção de cenário, modo custom)
│   │   ├── CandidateSetupScreen.tsx           # Tela de configuração de candidatos
│   │   └── ElectionSimulator.tsx              # Simulador principal (~113KB, componente gigante)
│   ├── CandidateManager.tsx                   # Gerenciador de candidatos dentro do simulador
│   ├── modals/                                # Modais (estado, foto, município, etc.)
│   └── photo/                                 # Componentes de geração de fotos
```

### Tipo `Candidate` (em `src/types.ts`):

```typescript
export interface Candidate {
  id: CandidateId;
  name: string;
  vice: string;
  party: string;
  number: string;
  color: string;
  photo?: string;
  vicePhoto?: string;
  partyLogo?: string;
  ideology?: string;
  coalition?: string;
  locked?: boolean;
}
```

### Tipo `PoliticalScenario` (em `src/types.ts`):

```typescript
export interface PoliticalScenario {
  id: string;
  name: string;
  year: number;
  round?: ElectionRound;
  description: string;
  candidates: Omit<Candidate, 'id'>[];
  results?: Record<string, Record<number, number>>;
  customStates?: CustomStateInfo[];
  nationalVoters?: number;
  isCustom?: boolean;
  generatedAt?: string;
}
```

### Fluxo do App (`src/App.tsx`):

1. Se `electionRound` é null → mostra `InitialScreen` (escolha de cenário)
2. Se `setupComplete` é false → mostra `CandidateSetupScreen` (configurar candidatos)
3. Se `setupComplete` é true → mostra `ElectionSimulator` (simulação)

`InitialScreen` pode chamar `handleSelectScenario(scenario)` que já pula direto para o simulador com os candidatos do cenário pré-carregados. Quando o usuário escolhe criar do zero (modo custom ou seleciona turno manualmente), vai para `CandidateSetupScreen`.
