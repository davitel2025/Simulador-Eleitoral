CORRECAO 1 — Bug: Botão "Alterar" no menu de contexto do município não funciona
═══════════════════════════════════════════════════════════════════

PROBLEMA:
No componente MunicipalityPaintModal.tsx, quando o usuário clica com o botão DIREITO do mouse em um município no mapa SVG, aparece um menu de contexto (contextMenu state). Esse menu exibe dados do município e tem um botão "Alterar" que deveria: (1) mudar o editMode para "percentage", (2) preencher o searchQuery com o nome do município, (3) fechar o menu. Mas atualmente ao clicar no "Alterar" NADA acontece.

CAUSA RAIZ DO BUG:
Há um useEffect que faz window.addEventListener("click", closeContextMenu) para fechar o menu quando o usuário clica em qualquer lugar. Quando o usuário clica no botão "Alterar", o evento de clique propaga para o window e o closeContextMenu é chamado, limpando o contextMenu state antes ou junto da execução do handler do botão. Isso anula a ação.

COMO CORRIGIR (passo a passo):
1. No useEffect que registra o listener de fechar o menu, troque "click" por "mousedown" para que o fechamento automático use mousedown ao invés de click. Isso resolve a colisão porque o onClick do botão dispara no mouseup/click, depois do mousedown.
2. No botão "Alterar" dentro do JSX do contextMenu, o handler deve ser:
   onClick={(event) => {
     event.stopPropagation();
     const nome = contextMenu.name; // salva antes de limpar
     setEditMode("percentage");
     setSearchQuery(nome);
     setContextMenu(null);
   }}
3. Certifique-se de que o elemento container do contextMenu que tem onClick={(event) => event.stopPropagation()} está funcionando corretamente para não propagar o clique dentro do menu para o window.
4. Após a correção, ao clicar em "Alterar": o mapa some, a lista de sliders de porcentagem aparece (editMode === "percentage"), e o campo de busca já está preenchido com o nome do município clicado, mostrando SOMENTE aquele município na lista.
5. Verifique também que o botão não está com pointer-events desabilitado por algum CSS herdado do container do mapa (o SVG container pode ter onContextMenu que captura eventos).
