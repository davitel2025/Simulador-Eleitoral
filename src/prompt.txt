## MODIFICAÇÃO 2 — Corrigir a navegação do mapa (zoom centrado no cursor, não no centro do Brasil)

### Contexto do problema
O mapa interativo do Brasil está em `src/components/screens/ElectionSimulator.tsx`. Atualmente o zoom com scroll do mouse **sempre** dá zoom em direção ao centro geométrico do mapa (centro do Brasil, que fica próximo ao Mato Grosso), independentemente de onde o cursor do usuário está. O resultado é que se o usuário quer dar zoom no Amazonas, o mapa puxa para o Mato Grosso, tornando a navegação ruim.

### Causa técnica
O código atual de scroll wheel zoom em `ElectionSimulator.tsx` faz algo assim (procure pelo `useEffect` que contém `onWheel`):

```ts
useEffect(() => {
  const mapNode = mapContainerRef.current;
  if (!mapNode) return;
  const onWheel = (event: WheelEvent) => {
    event.preventDefault();
    const delta = event.deltaY > 0 ? -0.12 : 0.12;
    setMapZoom((prev) => clamp(Number((prev + delta).toFixed(2)), 0.5, 5));
    // ← NÃO ajusta mapOffset baseado na posição do cursor
  };
  mapNode.addEventListener("wheel", onWheel, { passive: false });
  return () => mapNode.removeEventListener("wheel", onWheel);
}, []);
```

O zoom muda apenas a escala mas **não reposiciona** o offset para manter o ponto sob o cursor estático. Além disso, o `drag` do Framer Motion atualiza o `mapOffset` mas provavelmente com configuração que torna o arrasto pesado ou travado.

### O que fazer

#### 2.1 — Corrigir o zoom com scroll (zoom focado no cursor)

Substitua o `useEffect` do `onWheel` em `ElectionSimulator.tsx` por uma versão que calcula o novo offset para manter o ponto sob o cursor fixo ao aplicar o zoom. A lógica é:

```ts
useEffect(() => {
  const mapNode = mapContainerRef.current;
  if (!mapNode) return;

  const onWheel = (event: WheelEvent) => {
    event.preventDefault();

    const rect = mapNode.getBoundingClientRect();
    // Posição do cursor RELATIVA ao container do mapa
    const mouseX = event.clientX - rect.left;
    const mouseY = event.clientY - rect.top;

    const zoomFactor = event.deltaY > 0 ? -0.12 : 0.12;

    setMapZoom((prevZoom) => {
      const newZoom = clamp(
        Number((prevZoom + zoomFactor).toFixed(2)),
        0.5,
        5
      );
      const scale = newZoom / prevZoom;

      // Ajusta offset para manter o ponto sob o cursor fixo
      setMapOffset((prevOffset) => ({
        x: mouseX - scale * (mouseX - prevOffset.x),
        y: mouseY - scale * (mouseY - prevOffset.y),
      }));

      return newZoom;
    });
  };

  mapNode.addEventListener("wheel", onWheel, { passive: false });
  return () => mapNode.removeEventListener("wheel", onWheel);
}, []); // dependências vazias estão corretas pois usamos setters funcionais
```

**ATENÇÃO**: como o `setMapZoom` e `setMapOffset` são chamados dentro do callback funcional do `setMapZoom`, não haverá problema de stale closure. Mas confirme que `setMapOffset` é acessível dentro do callback — se necessário, use `useRef` para guardar os valores atuais de zoom e offset.

Se usar `useRef` para evitar stale closures, adicione:
```ts
const mapZoomRef = useRef(mapZoom);
const mapOffsetRef = useRef(mapOffset);

useEffect(() => { mapZoomRef.current = mapZoom; }, [mapZoom]);
useEffect(() => { mapOffsetRef.current = mapOffset; }, [mapOffset]);
```

E no `onWheel`:
```ts
const prevZoom = mapZoomRef.current;
const prevOffset = mapOffsetRef.current;
const newZoom = clamp(Number((prevZoom + zoomFactor).toFixed(2)), 0.5, 5);
const scale = newZoom / prevZoom;
setMapZoom(newZoom);
setMapOffset({
  x: mouseX - scale * (mouseX - prevOffset.x),
  y: mouseY - scale * (mouseY - prevOffset.y),
});
```

#### 2.2 — Suavizar o drag (arrasto do mapa)

Procure no JSX de `ElectionSimulator.tsx` onde o componente de drag do Framer Motion está sendo usado (provavelmente um `<motion.div>` ou `<motion.g>` com prop `drag` ou `onPan`/`onDrag`). O drag atual pode estar travado porque:

a) O Framer Motion `drag` tem inércia/constraints que travam, **OU**
b) O mapa está usando `onPan` do Framer Motion que acumula offsets de forma incorreta.

**Solução preferida** — substituir o arrasto do Framer Motion por eventos nativos de mouse/touch puros (mais confiável para SVG):

Adicione os seguintes estados e refs:
```ts
const isDraggingRef = useRef(false);
const lastMousePosRef = useRef({ x: 0, y: 0 });
```

No elemento container do mapa (o `div` que tem `ref={mapContainerRef}`), substitua as props de drag do Framer Motion por:
```tsx
onMouseDown={(e) => {
  isDraggingRef.current = true;
  lastMousePosRef.current = { x: e.clientX, y: e.clientY };
  e.preventDefault();
}}
onMouseMove={(e) => {
  if (!isDraggingRef.current) return;
  const dx = e.clientX - lastMousePosRef.current.x;
  const dy = e.clientY - lastMousePosRef.current.y;
  lastMousePosRef.current = { x: e.clientX, y: e.clientY };
  setMapOffset((prev) => ({ x: prev.x + dx, y: prev.y + dy }));
}}
onMouseUp={() => { isDraggingRef.current = false; }}
onMouseLeave={() => { isDraggingRef.current = false; }}
```

Adicione também suporte a touch para mobile:
```tsx
onTouchStart={(e) => {
  isDraggingRef.current = true;
  const touch = e.touches[0];
  lastMousePosRef.current = { x: touch.clientX, y: touch.clientY };
}}
onTouchMove={(e) => {
  if (!isDraggingRef.current) return;
  const touch = e.touches[0];
  const dx = touch.clientX - lastMousePosRef.current.x;
  const dy = touch.clientY - lastMousePosRef.current.y;
  lastMousePosRef.current = { x: touch.clientX, y: touch.clientY };
  setMapOffset((prev) => ({ x: prev.x + dx, y: prev.y + dy }));
}}
onTouchEnd={() => { isDraggingRef.current = false; }}
```

Se o mapa usa um `<motion.div>` com `onPan` do Framer Motion, remova as props `onPan`, `onPanStart`, `onPanEnd` e substitua pelo padrão acima com eventos nativos. Mantenha outros aspectos do Framer Motion nos estados dos estados (paths do SVG) se já existirem.

#### 2.3 — Garantir que o cursor muda de aparência

No container do mapa, adicione um estilo de cursor dinâmico para indicar que pode arrastar:
```tsx
style={{ cursor: isDragging ? "grabbing" : "grab" }}
```
Para isso, crie um estado `isDragging` (booleano) em vez de só o ref, ou use uma classe CSS condicional.

#### 2.4 — Botão de Reset de Posição

Se já existir um botão de "resetar zoom" no mapa, certifique-se de que ele também reseta o `mapOffset` para `{ x: 0, y: 0 }` junto com o `mapZoom` para `1`.

## MODIFICAÇÃO 3 — Corrigir o botão "Salvar PNG" em todos os modais de foto

### Contexto do problema
O botão "Salvar PNG" nos três modais (`NationalPhotoModal.tsx`, `RegionalPhotoModal.tsx` e `StatePhotoModal.tsx`) usa a função `captureAndDownload` (ou `savePhotoPng`) definida em `src/components/photo/PhotoCards.tsx`. Essa função usa a biblioteca `html2canvas` internamente.

Há **dois erros distintos** acontecendo:

#### Erro 1 — Imagens externas bloqueadas por CORS (OpaqueResponseBlocking)
```
GET https://upload.wikimedia.org/wikipedia/commons/thumb/e/e5/Mourão_foto_oficial_(cropped).jpg/440px-Mourão_foto_oficial_(cropped).jpg
NS_BINDING_ABORTED
A resource is blocked by OpaqueResponseBlocking
```
O html2canvas tenta buscar imagens externas (fotos de candidatos do Wikimedia, etc.) mas o navegador bloqueia por política de CORS. O `useCORS: true` e `allowTaint: true` já estão configurados, mas não são suficientes porque o servidor do Wikimedia não envia headers CORS permissivos para todas as requisições.

#### Erro 2 — Função de cor CSS não suportada (oklab)
```
Erro ao gerar PNG: Error: Attempting to parse an unsupported color function "oklab"
    parse color.ts:15
    parse index.ts:307
    CSSParsedDeclaration2 index.ts:161
    ElementContainer2 element-container.ts:27
```
O `html2canvas` não suporta funções de cor modernas como `oklab()`, `oklch()`, `color-mix()`, etc., que são geradas pelo Tailwind CSS v3.3+ e/ou por propriedades CSS modernas. O html2canvas tenta parsear os estilos computados e falha ao encontrar essas funções.

### Solução completa para ambos os erros

#### 3.1 — Resolver o erro de `oklab` (cor CSS não suportada)

A solução é substituir o `html2canvas` por **`html-to-image`**, uma biblioteca mais moderna que usa a API nativa do browser (`foreignObject` do SVG + `Canvas`) e suporta funções de cor modernas.

**Passo 1**: Instale a biblioteca:
```bash
npm install html-to-image
```

**Passo 2**: Abra `src/components/photo/PhotoCards.tsx` e localize a função `captureAndDownload`. Ela atualmente importa `html2canvas` dinamicamente. Substitua completamente pela implementação abaixo:

```ts
export async function captureAndDownload(
  element: HTMLElement,
  filename: string,
  backgroundColor: string
): Promise<void> {
  try {
    const { toPng } = await import("html-to-image");

    // Pré-carrega todas as imagens como base64 para evitar problemas de CORS
    await preloadImagesAsBase64(element);

    const dataUrl = await toPng(element, {
      backgroundColor,
      pixelRatio: Math.max(2, window.devicePixelRatio || 1),
      cacheBust: true,
      includeQueryParams: true,
      skipFonts: false,
      filter: (node) => {
        // Não pular nenhum elemento
        return true;
      },
    });

    const link = document.createElement("a");
    link.download = filename;
    link.href = dataUrl;
    document.body.appendChild(link);
    link.click();
    document.body.removeChild(link);
  } catch (error) {
    console.error("Erro ao gerar PNG:", error);
    alert(
      "Erro ao salvar imagem. Verifique o console para detalhes.\n\n" +
      "Dica: se o erro persistir, tente remover as fotos externas dos candidatos e usar apenas fotos enviadas por upload."
    );
  }
}
```

**Passo 3**: Adicione a função auxiliar `preloadImagesAsBase64` ANTES da função `captureAndDownload` em `PhotoCards.tsx`:

```ts
/**
 * Percorre o elemento e substitui temporariamente todos os src de <img>
 * e url() de CSS por versões base64, contornando o bloqueio de CORS.
 * Apenas tenta converter — se falhar (CORS total), mantém o original e prossegue.
 */
async function preloadImagesAsBase64(root: HTMLElement): Promise<void> {
  const imgs = Array.from(root.querySelectorAll<HTMLImageElement>("img"));

  await Promise.allSettled(
    imgs.map(async (img) => {
      const src = img.src;
      if (!src || src.startsWith("data:")) return; // já é base64, pula

      try {
        const base64 = await fetchImageAsBase64(src);
        img.src = base64;
      } catch {
        // silencia erros — mantém src original se não conseguir converter
      }
    })
  );
}

/**
 * Faz fetch de uma imagem e retorna como data URL base64.
 * Tenta com proxy CORS se a requisição direta falhar.
 */
async function fetchImageAsBase64(url: string): Promise<string> {
  // Tenta buscar diretamente (funciona para imagens no mesmo domínio ou com CORS)
  try {
    const response = await fetch(url, { mode: "cors" });
    if (response.ok) {
      const blob = await response.blob();
      return await blobToBase64(blob);
    }
  } catch {
    // falha silenciosa, tenta proxy
  }

  // Tenta via proxy CORS público (corsproxy.io)
  try {
    const proxiedUrl = `https://corsproxy.io/?${encodeURIComponent(url)}`;
    const response = await fetch(proxiedUrl);
    if (response.ok) {
      const blob = await response.blob();
      return await blobToBase64(blob);
    }
  } catch {
    // falha silenciosa
  }

  throw new Error(`Não foi possível carregar a imagem: ${url}`);
}

function blobToBase64(blob: Blob): Promise<string> {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = () => resolve(reader.result as string);
    reader.onerror = reject;
    reader.readAsDataURL(blob);
  });
}
```

#### 3.2 — Fallback com html2canvas caso html-to-image também falhe

Se por qualquer motivo o `html-to-image` também falhar (por exemplo, em browsers muito antigos), adicione um fallback para `html2canvas` com a configuração mais robusta possível:

```ts
// No bloco catch da função captureAndDownload, antes do alert:
try {
  // Fallback: tenta html2canvas
  const html2canvas = (await import("html2canvas")).default;
  const canvas = await html2canvas(element, {
    backgroundColor,
    scale: Math.max(2, window.devicePixelRatio || 1),
    useCORS: true,
    allowTaint: true,
    logging: false,
    imageTimeout: 15000,
    scrollX: 0,
    scrollY: -window.scrollY,
    windowWidth: document.documentElement.scrollWidth,
    windowHeight: document.documentElement.scrollHeight,
    onclone: (clonedDoc) => {
      // Remove estilos problemáticos que html2canvas não entende
      const allElements = clonedDoc.querySelectorAll("*");
      allElements.forEach((el) => {
        const htmlEl = el as HTMLElement;
        // Remove classes Tailwind que geram oklab/oklch
        htmlEl.style.colorScheme = "";
        // Substitui cores oklch/oklab por equivalentes hex se possível
        const computedStyle = window.getComputedStyle(htmlEl);
        const color = computedStyle.color;
        const bg = computedStyle.backgroundColor;
        if (color.includes("oklab") || color.includes("oklch")) {
          htmlEl.style.color = "#ffffff";
        }
        if (bg.includes("oklab") || bg.includes("oklch")) {
          htmlEl.style.backgroundColor = backgroundColor;
        }
      });
    },
  });
  const blob = await new Promise<Blob | null>((resolve) =>
    canvas.toBlob(resolve, "image/png")
  );
  if (blob) {
    const url = URL.createObjectURL(blob);
    const anchor = document.createElement("a");
    anchor.href = url;
    anchor.download = filename;
    document.body.appendChild(anchor);
    anchor.click();
    document.body.removeChild(anchor);
    setTimeout(() => URL.revokeObjectURL(url), 1000);
    return;
  }
} catch (fallbackError) {
  console.error("Fallback html2canvas também falhou:", fallbackError);
}
```

```tsx
function CandidateImage({ candidate, className, fallbackClassName = "text-3xl" }: ...) {
  const [hasImageError, setHasImageError] = useState(false);

  if (!candidate.photo || hasImageError) {
    return (
      <div className={`flex h-full w-full items-center justify-center font-black ${fallbackClassName}`}
           style={{ color: candidate.color }}>
        {getCandidateInitials(candidate)}
      </div>
    );
  }

  return (
    <img
      src={candidate.photo}
      alt={candidate.name}
      className={className}
      crossOrigin="anonymous"          // ← ADICIONAR
      referrerPolicy="no-referrer"     // ← ADICIONAR
      onError={() => setHasImageError(true)}
    />
  );
}
```
