CONTEXTO DO PROJETO:
Você tem acesso ao repositório local do projeto "Simulador Eleitoral" (https://github.com/davitel2025/Simulador-Eleitoral), que é uma aplicação React + TypeScript + Vite que simula resultados de eleições presidenciais brasileiras. O projeto usa D3-geo para mapas, Framer Motion para animações e Tailwind CSS para estilo. Ele já possui:
- Um mapa SVG interativo do Brasil dividido por estados (com GeoJSON do IBGE)
- Sistema de "pintura" de municípios por estado individualmente (MunicipalityPaintModal.tsx)
- Cenários históricos pré-definidos (historical scenarios) para eleições de 2018 e 2022
- Exportação/importação de cenários em JSON
- Geração de fotos/imagens (PNG) dos resultados nacionais, por estado e por região

Leia e analise TODOS os arquivos do projeto antes de fazer qualquer modificação, especialmente:
- src/components/screens/ElectionSimulator.tsx (arquivo principal, ~112KB — o maior do projeto)
- src/components/screens/CandidateSetupScreen.tsx
- src/components/modals/MunicipalityPaintModal.tsx
- src/components/modals/NationalPhotoModal.tsx
- src/components/modals/RegionalPhotoModal.tsx
- src/components/photo/PhotoCards.tsx
- src/data/ (todos os arquivos de dados, incluindo cenários históricos)
- Qualquer arquivo de tipos TypeScript (types.ts ou similar) que defina as interfaces dos cenários

TAREFA 3 — MAPA NACIONAL POR MUNICÍPIO (nova funcionalidade)
===========================================================================

Atualmente, no mapa principal (ElectionSimulator.tsx), o Brasil é exibido com estados
coloridos. Já existe a funcionalidade de "pintar municípios" dentro de cada estado
individualmente (MunicipalityPaintModal.tsx). A nova funcionalidade pedida é:

ADICIONAR UM BOTÃO/TOGGLE na interface do mapa principal (ElectionSimulator.tsx) que,
quando ativado, substitui a visualização de estados coloridos por uma visualização de
TODOS os municípios do Brasil coloridos simultaneamente no mapa nacional.

Especificações técnicas:

1. TOGGLE DE VISUALIZAÇÃO:
   - Adicione um botão ou switch na barra de controles do mapa (mesma área onde ficam
     os botões de zoom, legenda, etc.)
   - Sugestão de label: "Por município" / "Por estado" (alternando)
   - Ou use um ícone de mapa com tooltip explicativo
   - O toggle deve ser visualmente consistente com o design existente (dark theme, bordas
     arredondadas, estilo similar aos outros botões do projeto)

2. CARREGAMENTO DO GEOJSON DE MUNICÍPIOS:
   - O projeto provavelmente já carrega o GeoJSON de municípios via fetch assíncrono
     para o MunicipalityPaintModal. Verifique como isso é feito.
   - Reutilize a mesma fonte de dados de municípios, mas agora para renderizar todos
     os estados ao mesmo tempo no mapa nacional.
   - Se o GeoJSON de municípios for muito pesado para renderizar todos de uma vez,
     implemente lazy loading: carregue os municípios apenas quando o toggle for ativado,
     com um indicador de loading (spinner ou texto "Carregando municípios...").

3. RENDERIZAÇÃO NO SVG PRINCIPAL:
   - Quando o modo "por município" estiver ativo, substitua os paths dos estados pelos
     paths dos municípios dentro do mesmo SVG viewBox já definido no mapa.
   - Use a mesma projeção D3-geo já configurada no projeto.
   - Cada município deve ser colorido com a cor do candidato que o venceu (usando
     o campo municipalityData/municipalities do resultado de cada estado).
   - Se um município não tiver resultado definido, use a cor neutra já usada para
     estados sem resultado (cinza escuro ou a cor padrão do tema).
   - Municípios sem dado devem ser transparentes ou cinza, igual ao comportamento
     atual de estados não preenchidos.

4. INTERATIVIDADE NO MODO MUNICÍPIO:
   - Hover: ao passar o mouse sobre um município, exibir tooltip com:
     * Nome do município
     * Estado (UF)
     * Candidato vencedor e porcentagem (se disponível)
   - NÃO é necessário implementar clique para editar município no modo nacional
     (manter o comportamento atual: para editar, o usuário vai no modal por estado)
   - O zoom/pan já existente no mapa deve continuar funcionando normalmente

5. PERFORMANCE:
   - Se o GeoJSON de municípios for muito grande (pode ter milhares de paths SVG),
     considere simplificar as geometrias com turf.js ou D3's simplify, OU renderize
     apenas os municípios dos estados que já têm dados de municípios pintados.
   - Adicione um aviso visual se a performance estiver comprometida (ex: "Modo município
     pode ser lento em dispositivos mais simples").

6. ESTADO DO TOGGLE:
   - Persista o estado do toggle no estado local do componente (useState).
   - NÃO é necessário persistir no localStorage ou no JSON do cenário.
   - Quando o usuário alternar entre cenários, o toggle deve voltar para "por estado".
