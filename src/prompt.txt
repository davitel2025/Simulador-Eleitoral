Você é um engenheiro front-end sênior especializado em React, TypeScript, Tailwind CSS e Framer Motion. Você tem acesso completo ao meu repositório local do projeto "Simulador-Eleitoral" (https://github.com/davitel2025/Simulador-Eleitoral) e pode ler, editar e criar quaisquer arquivos necessários.

O projeto é um simulador de eleições presidenciais brasileiro, feito com React + TypeScript + Vite + Tailwind CSS + Framer Motion + D3-geo. Ele possui:
- Tela inicial com seleção de cenários históricos e personalizados
- Tela de configuração de candidatos
- Simulador principal com mapa interativo do Brasil, sidebar, modais de edição por estado/município, analytics e geração de imagens PNG

Sua tarefa é MELHORAR PROFUNDAMENTE a funcionalidade de "Alterar Porcentagens" dos estados (o modal/componente que aparece quando o usuário clica em um estado no mapa e escolhe "Editar" → que leva ao StateEditModal ou equivalente com sliders) E também melhorar o HUB/painel principal (ElectionSimulator.tsx e seus sub-componentes de sidebar e analytics). Faça tudo diretamente nos arquivos do projeto.

PARTE 1 — MELHORIAS NA FUNCIONALIDADE "ALTERAR PORCENTAGENS"
═══════════════════════════════════════════════════════════════

O modal de edição de porcentagens por estado (StateEditModal.tsx ou equivalente) deve ser completamente reformulado com as seguintes adições e melhorias:

1. SLIDER REDESENHADO (UX)
   - Mantenha os sliders existentes (um por candidato), mas redesenhe-os visualmente:
     * Trilha (track) do slider deve usar a cor do candidato (candidate.color) com gradiente e opacidade
     * O "thumb" (bolinha do slider) deve ser maior, com sombra colorida (box-shadow usando a cor do candidato) e animação suave de escala ao focar/arrastar (transform scale 1.15)
     * Exiba o valor atual (ex: "34,5%") em um tooltip flutuante acima do thumb enquanto o usuário arrasta, usando `useState` + `onMouseMove`/`onInput`
     * Use `input[type=range]` com estilização via Tailwind + CSS customizado (`::-webkit-slider-thumb` e `::-webkit-slider-runnable-track` via `@layer components` ou `style` inline)

2. CAMPO DE INPUT NUMÉRICO DIRETO
   - Ao lado de cada slider, adicione um campo `<input type="number" />` que:
     * Mostre o valor atual com 1 casa decimal
     * Permita ao usuário digitar diretamente um valor entre 0 e 100
     * Ao alterar o número, aplique a normalização automática nos outros candidatos (redistribuição proporcional)
     * Tenha borda colorida com a cor do candidato ao focar (`focus:ring` com a cor do `candidate.color`)
     * Tenha largura fixa de ~80px, centralizado e estilizado com bg-slate-800 e texto branco

3. BOTÕES DE PRESET RÁPIDO (Quick-fill)
   - Acima dos sliders, adicione uma faixa de botões de atalho com presets de distribuição:
     * "Empate técnico" → distribui 100% igualmente entre todos os candidatos
     * "Vantagem [NomeCandidato1]" → distribui 60% para o candidato 1 e o restante igualmente
     * "Vantagem [NomeCandidato2]" → igual, mas para o candidato 2 (e assim por diante, gere dinamicamente para cada candidato)
     * "Aleatório" → sorteia porcentagens aleatórias que somam 100%, usando Math.random() normalizado
     * "Limpar" → zera todos os campos e reseta para distribuição igual
   - Botões pequenos e elegantes com estilo `outline` (borda colorida, fundo transparente), fonte pequena (text-xs), com hover que preenche o fundo
   - Cada botão de "Vantagem X" deve ter um indicador circular com a cor do candidato à esquerda

4. BARRA DE TOTAL VISUAL (Live Sum Indicator)
   - Abaixo de todos os sliders, exiba uma barra horizontal que:
     * Mostra o total atual de todas as porcentagens somadas (deve sempre ser 100%)
     * Cada segmento da barra tem a largura proporcional à porcentagem e a cor do candidato
     * Tem animação suave via CSS `transition: width 0.3s ease`
     * Exibe o nome abreviado (ou número) de cada candidato dentro do segmento quando o segmento for largo o suficiente (> 10%)
     * Abaixo da barra, exibe "Total: 100,0%" em texto, com cor verde se exatamente 100, amarela se diferente (para casos de edição manual temporária)

5. MODO "TRAVAR CANDIDATO" (Lock)
   - Ao lado do nome de cada candidato no modal, adicione um ícone de cadeado (🔒/🔓) que, ao clicar:
     * Marca aquele candidato como "travado" — sua porcentagem não muda quando os outros sliders são movidos
     * A redistribuição proporcional ao mexer em um slider acontece SOMENTE nos candidatos não travados
     * Quando travado, o slider fica com `opacity: 0.5` e `pointer-events: none`
     * O cadeado ativo fica com cor amarela/âmbar, o inativo com cinza

6. HISTÓRICO DE ALTERAÇÕES COM UNDO (Ctrl+Z)
   - Implemente um histórico de até 10 estados anteriores usando `useRef` com um array de snapshots do objeto `votes`
   - Adicione um botão "↩ Desfazer" no modal, visível apenas quando há histórico disponível
   - Ao clicar em Desfazer, restaura o último estado salvo do objeto `votes` e atualiza os sliders
   - Use `useEffect` para registrar cada mudança no histórico com debounce de 300ms (para não salvar a cada pixel do slider)

7. COMPARAÇÃO COM MÉDIA NACIONAL (Benchmark Visual)
   - Se o cenário carregado (loadedScenario) tiver resultados preenchidos em outros estados, calcule a média nacional atual de cada candidato (já disponível no componente pai como `national.candidatePcts`)
   - No modal de cada estado, exiba abaixo do slider de cada candidato uma linha fina indicando a média nacional daquele candidato (uma marcação vertical na trilha do slider, tipo linha pontilhada)
   - Adicione um pequeno texto "Média nacional: X%" logo abaixo do slider, em cinza claro (text-slate-400, text-xs)

8. VISUALIZAÇÃO PRÉVIA DO MAPA DO ESTADO (Minimap Preview)
   - No topo do modal de edição, exiba um pequeno SVG do contorno geográfico do estado sendo editado (usando os dados já disponíveis em `stateGeoData` do componente pai, passando como prop para o modal)
   - Pinte esse mini SVG com a cor do candidato vencedor atual do estado (atualizada em tempo real conforme o usuário mexe nos sliders, antes de salvar)
   - Coloque o nome completo do estado e sua sigla UF abaixo do minimap

9. ANIMAÇÃO DE TRANSIÇÃO DO MODAL
   - O modal de edição deve usar `motion.div` do Framer Motion com:
     * `initial={{ opacity: 0, scale: 0.92, y: 20 }}`
     * `animate={{ opacity: 1, scale: 1, y: 0 }}`
     * `exit={{ opacity: 0, scale: 0.92, y: 20 }}`
     * `transition={{ duration: 0.22, ease: "easeOut" }}`
   - Use `AnimatePresence` ao redor do modal no componente pai

10. BOTÃO "APLICAR A TODOS OS ESTADOS DA REGIÃO"
    - No rodapé do modal de edição, adicione um botão secundário "Aplicar a toda a região [NomeDaRegião]"
    - Ao clicar, abre um pequeno popover/dialog de confirmação: "Deseja aplicar estas porcentagens para todos os estados da região X? Esta ação substituirá os dados existentes."
    - Com confirmação, itera sobre todos os estados da mesma região (use a constante STATES e o campo `region` de cada estado) e aplica o mesmo objeto `votes` do estado atual para todos eles via `setResults` no componente pai
    - Passe uma prop `onApplyToRegion: (votes: Record<number, number>, region: string) => void` do pai para o modal
