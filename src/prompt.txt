

Você tem acesso ao código completo do projeto Simulador Eleitoral localizado neste repositório. Seu objetivo é implementar todas as modificações listadas abaixo diretamente nos arquivos do projeto, sem pedir confirmação — edite, crie e corrija o que for necessário imediatamente.

O projeto é uma aplicação React + TypeScript + Vite + Tailwind CSS + D3-geo + html2canvas. Os principais arquivos relevantes estão em src/, incluindo src/data/presetCandidates.ts, src/data/scenarios.ts, src/data/historicalElectionResults.ts, src/components/screens/ElectionSimulator.tsx, e os componentes de geração de foto (PhotoGenerator e afins). Leia todos os arquivos necessários antes de começar a editar.

Atualmente, ao clicar em "Baixar PNG" nas fotos (nacional, regional, estadual), só existe um layout. Adicione uma tela/modal de seleção de formato antes do download, com duas opções:
Formato 16:9 (já existente)

    Mantém o layout horizontal atual, apenas garantindo que ele se adapte corretamente à proporção 16:9.
    Dimensões recomendadas para exportação: 1920×1080px (ou o equivalente já usado no projeto).

Formato 9:16 (novo — "layout de celular")

    Disponível apenas quando a eleição for de segundo turno (electionRound === "segundo"). Se for primeiro turno, essa opção deve aparecer desabilitada (cinza, com tooltip explicando que só está disponível no 2º turno).

    Dimensões para exportação: 1080×1920px.

    Layout vertical com os seguintes elementos, de cima para baixo, com fundo escuro (#0f172a ou similar) igual ao estilo já usado no projeto:

        Topo: Título centralizado pequeno (ex: "Resultado — 2º Turno 2026") em fonte pequena, com leve opacidade.

        Área superior/média (≈ 40% da altura): Os dois candidatos lado a lado ocupando a largura total.
            Cada candidato: card com foto em formato portrait (cantos arredondados, borda colorida com a cor do partido do candidato, sombra suave box-shadow).
            Abaixo da foto de cada candidato: nome em branco, fonte grossa (font-bold ou font-extrabold).
            Abaixo do nome: três pequenos blocos escuros arredondados (estilo badge/pill) contendo, respectivamente: vice, espectro ideológico (ideology) e slogan/coalizão (coalition). Fonte bem pequena.

        Área central (≈ 30% da altura): Mapa do Brasil centralizado, maior que no layout horizontal, com as mesmas cores de estados, contornos e estilo visual (flat, com sombra) já usados no projeto. Preservar transparência de fundo para que o fundo do card apareça.

        Área inferior (≈ 30% da altura): Percentuais e votos dos dois candidatos.
            O percentual do candidato da esquerda (candidato[0]) aparece no canto inferior esquerdo.
            O percentual do candidato da direita (candidato[1]) aparece no canto inferior direito.
            O percentual deve ser o maior elemento textual da composição (fonte muito grande, font-extrabold), colorido com a cor do partido de cada candidato.
            Abaixo do percentual: número de votos em texto pequeno, branco com opacidade reduzida.

    Todo o estilo visual (gradientes de fundo, tipografia, cores de borda, sombras) deve seguir o mesmo padrão já estabelecido no projeto nos cards de foto existentes.

Implementação técnica sugerida:

    Crie um componente separado, ex: VerticalPhotoCard.tsx (ou equivalente dentro da estrutura já existente de geração de foto do projeto).
    Use html2canvas (já instalado) para exportar o DOM renderizado em 1080×1920, da mesma forma que o layout horizontal já faz.
    O modal de seleção pode ser um simples overlay/dialog com dois botões: "16:9 (Horizontal)" e "9:16 (Celular)". Se for primeiro turno, o botão 9:16 fica desabilitado.

MODIFICAÇÃO 3 — Corrigir bug de porcentagens iguais ao alterar municípios e ao importar cenário de 2018

Bug reportado: Quando o usuário está na tela de "alterar por município" (tela MunicipalityPainterScreen ou o dialog equivalente dentro de ElectionSimulator), ao abrir um estado e editar as porcentagens municipais, todos os municípios mostram a mesma porcentagem para todos os candidatos, e apenas um candidato aparece como vencedor em todos os municípios, independente dos valores reais.

Causa provável: A função que lê os votos municipais (municipalities dentro de StateResult) ou a função que calcula o vencedor por município (municipalityPaint) está usando índices de candidato incorretos (provavelmente sempre 1 ao invés do ID real do candidato) ou está lendo o campo errado do estado. Verifique especialmente:

    A função buildHistoricalStateResults em src/data/historicalElectionResults.ts — ela mapeia candidatos pelo index + 1 como chave de ID, mas os dados históricos de município usam o número eleitoral do candidato como chave. A função getHistoricalMunicipalityCandidatePcts converte corretamente por candidate.number, mas verifique se o resultado é aplicado ao StateResult.municipalities com as chaves corretas (IDs dos candidatos do cenário atual, não números eleitorais).
    No componente de pintura de municípios, verifique se ao calcular winnerPct e winnerIndex para exibição, o código está buscando pelo candidate.id correto e não pelo index ou number.
    Ao importar um cenário histórico (ex: "Eleições 2018"), os candidatos recebem id: index + 1 em App.tsx (handleSelectScenario). Confirme que esse ID bate com as chaves usadas em StateResult.votes e StateResult.municipalities.

Corrija o bug garantindo que:

    Ao carregar um cenário histórico com municipalityResultStrategy, os dados municipais são corretamente carregados com as chaves de ID dos candidatos do cenário atual.
    A exibição no mapa de municípios mostra corretamente quem venceu cada município com a porcentagem real.
    Ao editar manualmente a porcentagem de um município via slider, o valor é salvo e exibido corretamente, sem sobrescrever com valores iguais para todos.

        Botão/opção clicável: "✏️ Alterar porcentagem" — ao clicar, abre um mini-dialog/modal inline (pode ser o mesmo padrão visual já usado no projeto) onde o usuário pode ajustar a porcentagem de cada candidato naquele município via slider ou input numérico, e confirmar.
    O menu deve fechar ao clicar em qualquer outro lugar da tela (mesmo comportamento do contextMenu de estados).

Implementação:

    Crie um estado municipalityContextMenu (análogo ao contextMenu já existente para estados) com campos: { uf: string, code: string, name: string, x: number, y: number } | null.
    Adicione o onContextMenu nos paths de município dentro do SVG (na seção mapMunicipalityMode ? activeMunicipalityPaths.map(...) do componente).
    Renderize o menu flutuante no mesmo container que já renderiza o contextMenu dos estados, logo abaixo (ou em paralelo).
    O mini-dialog de edição de porcentagem por município deve salvar os valores no results[uf].municipalities[code] e recalcular o municipalityPaint[code] com base no candidato de maior porcentagem.

MODIFICAÇÃO 5 — Corrigir imagens CORS dos candidatos de 2018 (e logos de partidos)

Problema: Vários candidatos e logos de partidos usam URLs do commons.wikimedia.org/wiki/Special:FilePath/... via a função commonsFile() em src/data/presetCandidates.ts. Essas URLs fazem um redirect 302 para o arquivo real, mas o browser bloqueia o redirect por política CORS, resultando nos erros documentados em src/erro.txt.

Candidatos/logos afetados (todos usando commonsFile()):

    marinaSilva → commonsFile("Marina_Silva_2023.jpg")
    caboDaciolo → commonsFile("Cabo_Daciolo.jpg")
    PARTY_LOGOS.patri → commonsFile("Patriota_51_logo.png")
    PARTY_LOGOS.rede → commonsFile("Rede_Sustentabilidade_logo.svg")
    E todos os outros que usam commonsFile(): henriqueMeirelles, veraLucia, alvaroDias, constituinteEymael, guilhermeBoulos, joaoGoulartFilho, leoPericles, PARTY_LOGOS.pstu, PARTY_LOGOS.pode, PARTY_LOGOS.dc, PARTY_LOGOS.psol, PARTY_LOGOS.ppl.

Solução: Substitua todas as chamadas commonsFile(filename) por URLs diretas do Wikimedia que não usam redirect e têm CORS liberado. O padrão correto é:

text

https://upload.wikimedia.org/wikipedia/commons/thumb/[A]/[AB]/[NomeDoArquivo]/500px-[NomeDoArquivo]

ou para arquivos sem miniatura (PNG/JPG diretos não-SVG):

text

https://upload.wikimedia.org/wikipedia/commons/[A]/[AB]/[NomeDoArquivo]

Para cada arquivo afetado, encontre a URL direta correta do Wikimedia (sem /wiki/Special:FilePath/). Use as seguintes URLs diretas substitutas:
